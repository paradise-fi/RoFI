bool computeMatrices()

Computes matrices for all shoes.

This method is called in method isValid.

User should call this if they create new configuration or they change some angles and they don’t call method isValid.

Returns

false if there is inconsistency in computing matrices, i.e. matrix for one shoe was computed two different ways and the results aren’t equal (rotated module, that couldn’t be rotated)

Matrix computeConnectedMatrix(Edge edge) const

Computes matrix for shoe at the end of edge from shoe at the begining of edge.

For example, module with id = 1 is connected from shoe A to shoe B of module with id = 2. It takes matrix of shoe A of module 1 and computes matrix of shoe B of module 2 from that.

If neither matrix of shoes of the first module is computed, error is thrown.

Returns

Matrix of the shoe connected to the end of edge

Matrix computeOtherSideMatrix(ID id, ShoeId side) const

Given id and side computes matrix for the other shoe of the given module.

Returns

bool execute(const Action &action)

Tries to execute given action.

False is returned if either you try to add edge to already occupied dock, erase non-existent edge, perform invalid rotation or configuration wasn’t connected before executing.

This function does NOT check if two modules collide during rotations. For this functionality use executeIfValid from Generators.h

Returns

True if action was executed without problems.
